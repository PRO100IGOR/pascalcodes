环境及升级设置

在 CnPack 菜单中选择“设置”项，专家弹出“CnPack IDE 专家设置”界面，“环境设置”页面如下图所示：

　

提示信息

  • 允许专家第一次执行时显示功能提示窗口： 是否允许专家工具在第一次执行时显示提示信息，如果关闭则所有提示窗口都将不显示。见 IDE 专家基本操作。

  • 允许显示工具提示信息：是否允许显示控件 Hint 提示信息。

  • 重新开启所有提示信息：点击该按钮后，所有提示信息设置被重置，您在执行专家工具时，会重新获得提示。

　

升级设置

  • 关闭升级提示功能：关闭 IDE 启动时自动检测最新升级的功能。

  • 在有新版本可用时显示提示：允许专家包在检测到新的升级程序时进行提示。

  • 检测时忽略测试版的程序：如果选中，只有当新的升级不是测试版时才显示升级提示。

  • 立即检测最新的版本：从服务器上获得最新的版本信息，包括测试版程序。

请查看 CnPack 开发包升级提示。

　

辅助工具

  • 导出 IDE 图像列表位图：将 IDE 主图像列表保存到指定目录下，主要供开发人员使用。

  • 导出 IDE ActionList 名称：保存 IDE 主 ActionList 内容到指定文件中，主要供开发人员使用。

  • 导出已安装控件列表：将 IDE 中已安装的包和控件列表分别保存到 Packages.txt 和 Components.txt 文件中，主要供开发人员使用。控件列表中可能因为 VCL 和 CLX 包的并存而存在重复的控件名。

　

其他

  • 将 CnPack 专家菜单置于 Tools 菜单下：设置将“CnPack”菜单项放在主菜单项中还是置于“Tools”菜单下。

  • 设置 IDE 只在 CPU 0 单核中运行：用于解决某些多核环境下可能导致 IDE 不稳定的问题。

  • 使用 SetThreadLocale 修正 Vista / Win7 下乱码问题：修正某些 IDE 在 Vista / Win7 下中文显示为乱码的问题。

  • 指定用户数据目录位置: 该选项允许将用户数据放置到用户文档目录下，可解决 Vista/Windows 7 下运行需要管理权限的问题。
