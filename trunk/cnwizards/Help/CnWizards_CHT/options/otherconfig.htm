環境及升級設置

在 CnPack 功能表中選擇“設置”項，專家彈出“CnPack IDE 專家設置”介面，“環境設置”頁面如下圖所示：

　

提示資訊

  • 允許專家第一次執行時顯示功能提示視窗： 是否允許專家工具在第一次執行時顯示提示資訊，如果關閉則所有提示視窗都將不顯示。見 IDE 專家基本操作。

  • 允許顯示工具提示資訊：是否允許顯示控制項 Hint 提示資訊。

  • 重新開啟所有提示資訊：點擊該按鈕後，所有提示資訊設置被重置，您在執行專家工具時，會重新獲得提示。

　

升級設置

  • 關閉升級提示功能：關閉 IDE 啟動時自動檢測最新升級的功能。

  • 在有新版本可用時顯示提示：允許專家包在檢測到新的升級程式時進行提示。

  • 檢測時忽略測試版的程式：如果選中，只有當新的升級不是測試版時才顯示升級提示。

  • 立即檢測最新的版本：從伺服器上獲得最新的版本資訊，包括測試版程式。

請查看 CnPack 開發包升級提示。

　

輔助工具

  • 匯出 IDE 圖像清單點陣圖：將 IDE 主圖像清單保存到指定目錄下，主要供開發人員使用。

  • 匯出 IDE ActionList 名稱：保存 IDE 主 ActionList 內容到指定檔中，主要供開發人員使用。

  • 匯出已安裝控制項清單：將 IDE 中已安裝的包和控制項清單分別保存到 Packages.txt 和 Components.txt 檔中，主要供開發人員使用。控制項清單中可能因為 VCL 和 CLX 包的並存而存在重複的控制項名。

　

其他

  • 將 CnPack 專家功能表置於 Tools 功能表下：設置將“CnPack”功能表項目放在主功能表項目中還是置於“Tools”菜單下。

  • 設置 IDE 只在 CPU 0 單核中運行：用於解決某些多核環境下可能導致 IDE 不穩定的問題。

  • 使用 SetThreadLocale 修正 Vista / Win7 下亂碼問題：修正某些 IDE 在 Vista / Win7 下中文顯示為亂碼的問題。

  • 指定使用者資料目錄位置: 該選項允許將使用者資料放置到使用者文檔目錄下，可解決 Vista/Windows 7 下運行需要管理許可權的問題。
