INI 讀寫單元生成嚮導

該嚮導可以根據一現成的 INI 文件生成一讀寫此 INI 文件的單元，目前支持 Pascal 和 C++。執行此嚮導後，顯示的窗體如圖所示：

  • INI 文件：打開一個現成的 INI 文件，此項必須選擇一個存在的文件。

  • 常量前綴：INI 讀寫單元中會生成一系列常量字符串來表示該 INI 文件中的 Section 和 Names 等。Section 常量的命名採用「常量前綴 + Section 名稱」的方式，各個 Section 中的 Name 的命名採用「常量前綴 + Name 所在的 Section 名稱 + Name 名稱」的方式，本項即用來設置此常量前綴，默認為「csIni」。

  • 類名：INI 讀寫單元中會生成一個類和此類的實例作為讀寫 INI 文件的對象，此項即用來設置此類的類名，無需輸入類名前綴的 T，默認值是「IniOptions」。

  • 自動判斷類型：如果勾上此選項，專家將自動判斷 INI 文件中各項的類型，否則一律以字符串來對待。目前能自動判斷的類型包括String, Integer, Float, DateTime, Boolean等幾種。

  • 將 0 和 1 作為布爾類型來處理：如果勾上此選項，專家將自動將 INI 文件中值為 0 或 1 的項作為布爾值，否則以整數來對待。

該專家在 Delphi 和 C++Builder 下都有效。在 BDS 2006 中，該嚮導能自動判斷需要生成的語言類型，如無法獲得當前語言類型，則提示用戶選擇。在 BDS 2005/2006 中，生成單元前需要預先選擇保存的單元名。
