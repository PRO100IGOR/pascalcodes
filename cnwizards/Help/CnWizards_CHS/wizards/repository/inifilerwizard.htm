INI 读写单元生成向导

该向导可以根据一现成的 INI 文件生成一读写此 INI 文件的单元，目前支持 Pascal 和 C++。执行此向导后，显示的窗体如图所示：

  • INI 文件：打开一个现成的 INI 文件，此项必须选择一个存在的文件。

  • 常量前缀：INI 读写单元中会生成一系列常量字符串来表示该 INI 文件中的 Section 和 Names 等。Section 常量的命名采用“常量前缀 + Section 名称”的方式，各个 Section 中的 Name 的命名采用“常量前缀 + Name 所在的 Section 名称 + Name 名称”的方式，本项即用来设置此常量前缀，默认为“csIni”。

  • 类名：INI 读写单元中会生成一个类和此类的实例作为读写 INI 文件的对象，此项即用来设置此类的类名，无需输入类名前缀的 T，默认值是“IniOptions”。

  • 自动判断类型：如果勾上此选项，专家将自动判断 INI 文件中各项的类型，否则一律以字符串来对待。目前能自动判断的类型包括String, Integer, Float, DateTime, Boolean等几种。

  • 将 0 和 1 作为布尔类型来处理：如果勾上此选项，专家将自动将 INI 文件中值为 0 或 1 的项作为布尔值，否则以整数来对待。

该专家在 Delphi 和 C++Builder 下都有效。在 BDS 2006 中，该向导能自动判断需要生成的语言类型，如无法获得当前语言类型，则提示用户选择。在 BDS 2005/2006 中，生成单元前需要预先选择保存的单元名。
