1. 每个向服务器发起的http请求都要追加mobile=true，方便session过滤器根据这个参数做出正确的响应。
  2. oxhide的菜单增加属性：移动客户端对应页面、移动图标。默认情况下，菜单点击后访问的http服务为： 原有菜单bs的访问地址+mobile=1,baseshowaction中根据mobile=1调用不同的方法，如showview时调用nextShowMobileView或者nextShowView。
  3. list请求自动包含being（起始记录）、count（查询条数）。查询条件必须是名为condition的数组，数组中元素有name、c（关系）、value三个属性,所有发送到服务端的请求参数名默认都是data
  4. 所有的http请求返回{message:""}表示错误信息，此时会弹出错误信息而不会执行http请求回调方法
